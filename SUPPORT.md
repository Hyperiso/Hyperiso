# Support

Use GitHub Issues for bug reports, feature requests and documentation problems.

Before opening an issue:

1. Search existing issues.
2. Check the README and relevant examples.
3. Include a minimal input file whenever the problem depends on LHA/SLHA/FLHA data.
4. State whether the issue affects C++, Python, CLI or Dash GUI.

For scientific validation questions, use the `Physics validation` issue template and include:

- model and backend;
- observable or Wilson coefficient;
- input parameters;
- expected result and source;
- HyperIso result;
- tolerance or precision requirement.

For security reports, follow `SECURITY.md` instead of opening a public issue.
